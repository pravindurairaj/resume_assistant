# ATS-Friendly Resume Guidelines

## Formatting Rules

1. **No tables, columns, or text boxes** — ATS parsers read left-to-right, top-to-bottom
2. **No headers/footers** — content in these areas is often skipped
3. **No images, icons, or graphics** — they are invisible to ATS
4. **Standard section headings** — use exact labels: "Work Experience", "Education", "Skills", "Certifications", "Projects", "Summary"
5. **Simple bullet points** — use standard dash or bullet characters
6. **Consistent date format** — use `MMM YYYY – MMM YYYY` (e.g., Jan 2020 – Mar 2023)
7. **Standard fonts** — Arial, Calibri, Times New Roman, or their equivalents
8. **No abbreviations without full form** — write "Search Engine Optimization (SEO)" on first use
9. **File format** — PDF (from text, not scanned image) or .docx

## Content Rules

1. **Keyword matching** — mirror exact phrases from the job description (e.g., if JD says "CI/CD pipelines", use "CI/CD pipelines" not "continuous integration")
2. **Quantify achievements** — use numbers, percentages, dollar amounts where possible
3. **Action verbs** — start each bullet with a strong action verb (Led, Designed, Implemented, Optimized, etc.)
4. **No personal pronouns** — avoid "I", "me", "my"
5. **Relevance ordering** — list most JD-relevant skills and experience first
6. **No filler words** — remove "responsible for", "duties included"; go straight to the verb
7. **Spell out acronyms once** — then use the acronym: "Amazon Web Services (AWS)"

## Section Order (Recommended)

1. **Name & Contact Info** — full name, phone, email, LinkedIn, location (city, state)
2. **Professional Summary** — 2-3 sentences tailored to the target role
3. **Skills** — grouped into categories, JD-matched skills listed first
4. **Work Experience** — reverse chronological, 3-5 bullets per role
5. **Projects** — if relevant to the JD
6. **Education** — degree, institution, graduation date
7. **Certifications** — relevant certs only

## Bullet Point Formula

**Action Verb + What You Did + How/With What + Measurable Result**

Examples:
- Designed and implemented a microservices architecture using Spring Boot, reducing API response time by 40%
- Led a team of 5 engineers to deliver a real-time data pipeline processing 2M+ events/day
- Automated CI/CD workflows using Jenkins and Docker, cutting deployment time from 2 hours to 15 minutes

## Keyword Matching Strategy

1. Extract hard skills (tools, languages, frameworks) from JD
2. Extract soft skills and domain terms from JD
3. Map each extracted keyword to the candidate's actual experience
4. **ONLY use keywords the candidate genuinely possesses** — never fabricate
5. Place matched keywords in: Summary, Skills section, and within Work Experience bullets
6. Use both the acronym and full form for maximum matching
