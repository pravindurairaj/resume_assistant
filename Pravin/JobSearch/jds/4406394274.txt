Role overview
As a Senior Test Automation Engineer on our TrustX SaaS platform, you will design, develop, and maintain automated tests that ensure the reliability, scalability, and correctness of our identity and authentication systems. You’ll work closely with backend, frontend, mobile, DevOps, architects, and service delivery teams to validate end‑to‑end system behavior across multiple environments.
This role is ideal for a strong engineer who understands how distributed systems fit together - not someone limited to UI testing. You should be able to design integration tests, API tests, automation workflows, data‑driven tests, and environment‑aware validation routines for a cloud‑native SaaS product.
You will also be expected to use AI‑powered tools such as GitHub Copilot, AI-driven test generation, and automated test analysis to increase speed, coverage, and efficiency.
What you will do
:
Automation Framework & Test Development
Design, build, and enhance automated test frameworks covering API, UI, integration, performance, and regression testing.
Develop automated test suites using modern tools such as JUnit, TestNG, Selenium, Cypress, Playwright, RestAssured, or similar.
Write tests that validate distributed systems, multi-step workflows, asynchronous operations, and cross‑service interactions.
Collaborate with backend, frontend, and mobile engineers to understand system flows and build meaningful end‑to‑end test coverage.
Validate API contracts, authentication flows, session behavior, and data integrity across microservices.
Ensure tests work across dev, QC, staging, and production-like environments.
Use GitHub Copilot and other AI tools to generate test cases, automation scaffolding, and documentation.
Adopt AI-powered techniques for identifying missing coverage, edge‑case exploration, and regression analysis.
Help the QA team establish best practices for AI‑assisted test creation and maintenance.
Build automated health checks and smoke tests for release pipelines.
Work with DevOps to instrument logs, metrics, and traces to support test diagnostics and troubleshoot failures.
Participate in performance and load testing efforts, analyzing bottlenecks and system-level issues.
Work closely with service delivery managers to validate release readiness and drive quality gates for deployments.
Collaborate with architects and developers on design approaches that improve testability and maintainability.
Support P1 incident investigations through automated reproduction, diagnostics, and post‑incident test coverage improvements.
Participate in code reviews and contribute to best coding practices within the QA automation group.
